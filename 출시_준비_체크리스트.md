## 4. 서버 및 도메인 준비

### 도메인 구매
- [ ] 도메인 이름 결정
  - 예: `sajuapp.com`, `jaehoe-saju.com`
- [ ] 가비아/후이즈에서 구매
- [ ] DNS 설정 (AWS Lightsail IP 연결)

### SSL 인증서
- [ ] Let's Encrypt 설치
- [ ] 자동 갱신 설정

### 서버 보안 설정
- [ ] 방화벽 설정 (UFW)
```bash
sudo ufw allow 22    # SSH
sudo ufw allow 80    # HTTP
sudo ufw allow 443   # HTTPS
sudo ufw allow 3306  # MySQL (localhost만)
sudo ufw enable
```

- [ ] Fail2ban 설치 (무차별 대입 공격 방어)
```bash
sudo apt install fail2ban
```

- [ ] SSH 키 인증 설정 (비밀번호 로그인 비활성화)

### 백업 설정
- [ ] 데이터베이스 자동 백업 스크립트
```bash
# /home/ubuntu/backup.sh
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u sajuadmin -p사주_app > /home/ubuntu/backups/saju_$DATE.sql
# 30일 이상 된 백업 삭제
find /home/ubuntu/backups -name "*.sql" -mtime +30 -delete
```

- [ ] Cron으로 매일 새벽 3시 실행
```bash
crontab -e
0 3 * * * /home/ubuntu/backup.sh
```

## 11. 위험 관리

### 기술적 위험
- **서버 다운**: 모니터링 + 자동 재시작 설정
- **결제 오류**: 토스페이먼츠 테스트 환경 충분히 검증
- **데이터 손실**: 자동 백업 + 복구 절차 문서화

### 법적 위험
- **개인정보 유출**: 암호화 + 최소한의 정보만 수집
- **사주 정확도 클레임**: 면책 조항 명시 ("참고용")
- **환불 요청**: 명확한 환불 정책 + 빠른 대응

### 재무적 위험
- **초기 수익 부족**: 마케팅 예산 단계적 집행
- **운영비 증가**: 사용자 증가 시 서버 업그레이드 계획

## 14. 긴급 대응 계획

### 서버 다운 시
1. PM2에서 자동 재시작 (설정됨)
2. 재시작 안 되면 수동 재시작
```bash

```
3. 로그 확인
